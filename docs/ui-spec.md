# LazyDay — Design Spec v1

Две светлые темы, переключаемые по времени суток, + тёмная тема как план Б.
Философия: «ленивый» дизайн — пользователь получает результат за 0–2 тапа.

---

## 1. Ленивые принципы (UX-инварианты)

1. **0 тапов до ленты** — профиль в localStorage, лента грузится сразу при открытии.
2. **1 тап = смена настроения** — mood-пресеты в один ряд над лентой, это главный орган управления.
3. **2 тапа максимум** до любого действия: сохранить, скрыть, открыть детали, билеты.
4. **Ничего не спрашивать повторно** — компания, питомец, интересы спрашиваются один раз в онбординге.
5. **Зона большого пальца** — все интерактивные элементы в нижних 2/3 экрана; tab bar снизу, фильтры — bottom sheet.
6. **Тач-таргеты ≥ 48×48 px** (чипы визуально меньше, но hit-area добивается padding'ом).
7. **Один accent CTA на экран** — остальные кнопки secondary/ghost.
8. **Скелетоны вместо спиннеров** — форма карточки видна до загрузки.

## 2. Логика тем

| Тема | Когда | Класс на `<html>` |
|---|---|---|
| День «Мёд и фисташка» | 06:00–17:59 локального времени города | `theme-day` |
| Вечер «Тбилиси после заката» | 18:00–05:59 | `theme-evening` |
| Тёмная (план Б) | ручной выбор в настройках или `prefers-color-scheme: dark` при `auto` | `theme-dark` |

Правила:

- Переключение день↔вечер автоматическое, по таймзоне города (`CityConfig.timezone`), не по устройству.
- В настройках селектор: **Авто (день/вечер)** · Всегда день · Всегда вечер · Тёмная. Хранить в `ProfileStore.theme` (localStorage).
- Смена темы — CSS transition 400 ms на `background-color` и `color` корневых поверхностей (уважать `prefers-reduced-motion`).
- Проверка времени: при старте + при возврате в foreground (`visibilitychange`), без интервал-таймеров.
- Вся вёрстка одна; темы = наборы CSS custom properties. Ни один компонент не знает про тему.

```css
  --ld-open: #2E7D4F;  --ld-open-soft: #E1F0E4;  --ld-on-open-soft: #1F5C36;
```


Вечерние отличия в поведении (не только цвет):

- Активный mood-чип — **solid** заливка primary (днём — soft-заливка). Смелее.
- Приветствие в шапке окрашено в `--ld-primary` (днём — обычный текст).
- Допустим один градиент на экран: подложка greeting-блока `linear-gradient(135deg, #E3455C 0%, #6E52C6 100%)` с прозрачностью 8–10 % поверх `--ld-bg`. Больше нигде градиенты не используем.
- События вечером поднимаются в выдаче — это скоринг, но визуально событийные карточки получают чуть более заметную полосу (5 px vs 4 px днём).

## 4. Типографика

| Роль | Шрифт | Размер / вес | Где |
|---|---|---|---|
| Display | **Unbounded** 500 | 22–24 px | Приветствие на Discover, логотип, заголовки онбординга |
| Заголовок карточки | **Manrope** 700 | 17 px / lh 1.3 | title места/события |
| Заголовок страницы | Manrope 700 | 20 px | Detail, Saved, Settings |
| Body | Manrope 400 | 15 px / lh 1.5 | описания, детальная страница |
| Метаданные | Manrope 400 | 13 px | «Парк · 1.8 км · 23 мин» |
| Бейджи/чипы | Manrope 500 | 12–13 px | explanations, статусы |
| Caption | Manrope 400 | 12 px | подписи табов, дата |

- Fallback-стек: `'Manrope', 'Noto Sans Georgian', system-ui, sans-serif`. Unbounded имеет кириллицу, но не грузинский — для `ka` display-роль падает в `'Noto Sans Georgian'` 700.
- Unbounded используем **дозированно**: максимум один display-элемент на экран. Весь UI — Manrope.
- Минимальный размер текста: 12 px. Числа (рейтинг, расстояние) — tabular-nums.

## 5. Геометрия, тени, движение

| Токен | Значение |
|---|---|
| Базовая сетка | 4 px; отступы контента страницы 16 px |
| Радиусы | карточки **20 px**, кнопки 14 px, чипы/бейджи 999 px, bottom sheet 24 px (верх), инпуты 12 px |
| Тень карточки | `0 4px 16px rgba(50, 40, 20, 0.06)` (день) / `rgba(30, 20, 40, 0.08)` (вечер) / нет теней в тёмной — только бордер |
| Тень sheet | `0 -8px 28px rgba(0,0,0,0.10)` |
| Высота tap-таргетов | кнопки 48 px, чипы 36 px визуально + hit-area 44 px, табы 56 px |
| Движение | 150 ms `ease-out` (нажатия), 250 ms (sheet, появление карточек), 400 ms (смена темы). `prefers-reduced-motion` → всё до 0 |
| Нажатие карточки/кнопки | `transform: scale(0.98)` |
| Появление ленты | stagger-fade первых 5 карточек, 40 ms шаг; без параллаксов |

## 6. Компоненты

### 6.1 App shell / Tab bar
- 3 таба: Лента (`ti ti-compass`), Избранное (`ti ti-heart`), Профиль (`ti ti-user`).
- Фон `--ld-surface`, верхний бордер `--ld-border`, safe-area-inset снизу.
- Активный таб: иконка + подпись `--ld-primary`, вес 500. Неактивный: `--ld-text-3`.
- Без лейблов нельзя — подписи 11–12 px обязательны (3 таба, места хватает).

### 6.2 Greeting header (Discover)
- Строка контекста 12 px `--ld-text-2`: «суббота · Тбилиси».
- Display-фраза Unbounded 22 px, меняется по времени: утро «Доброе утро. Куда лениво сходить?» / день «Лениво? Сейчас найдём.» / вечер «Куда выйдем вечером?» / ночь «Не спится? Есть варианты.» Копия локализуется (ru/en/ka).
- Вечером фраза цвета `--ld-primary`.
- Header скроллится вместе с лентой (не sticky) — sticky остаётся только context bar.

### 6.3 Context bar (sticky)
- Один ряд пилюль на `--ld-bg` (полупрозрачный blur не нужен — solid фон): [📍 локация · радиус] [время]. Иконки `--ld-primary`.
- Тап по пилюле → bottom sheet соответствующей настройки. Никаких инлайн-редакторов.
- Высота 44 px, при скролле прижимается под status bar.

### 6.4 Mood-пресеты
- Горизонтальный scroll-ряд чипов сразу под context bar. Это **главный CTA экрана**.
- Чип: иконка Tabler 16 px + слово. Один активный максимум; повторный тап снимает.
- День: активный = `--ld-primary-soft` + текст `--ld-on-primary-soft`; вечер: активный = solid `--ld-primary` + `--ld-on-primary`.
- Набор чипов зависит от времени: день — Прогулка, Кофе, С детьми, Активно, Поесть, Культура; вечер — Выйти, Бар, Музыка, Ужин, Кино, Романтика. (Маппинг на существующие 6 mood-пресетов + интересы.)
- Применение пресета = мгновенный перезапрос ленты, скролл к началу, лёгкий haptic (если PWA позволяет).

### 6.5 Result card — место
Анатомия сверху вниз:
1. Ряд: **title** 17/700 (1 строка, ellipsis) + иконка сердца 20 px справа (`--ld-text-3` → `--ld-heart` заливкой при сохранении, анимация scale 1→1.2→1).
2. Метаданные 13 px `--ld-text-2`: «Категория · 1.8 км · 23 мин пешком».
3. Ряд бейджей (max 3 explanation + 1 статус, wrap на 2 строки максимум):
    - Статус: «Открыто до 20:00» `--ld-open-soft`; «Закрыто» `--ld-closed-soft`; «Закроется через 40 мин» `--ld-glow-soft`(вечер)/warn-тон(день).
    - Интерес: «Вы любите: природа» `--ld-primary-soft`.
    - Контекст: «Pet friendly», «Для детей» `--ld-secondary-soft`.
4. Рейтинг: `ti ti-star` цвета `--ld-warn` + «4.8» 13/500 + «(2 741)» `--ld-text-2`.
- Карточка: `--ld-surface`, радиус 20, padding 14–16, бордер 1 px `--ld-border` + тень.
- Тап по карточке → detail. Swipe влево → действие «Скрыть» (фон `--ld-danger` soft, иконка `ti ti-eye-off`). Без фото в MVP — карточка живёт текстом и бейджами.

### 6.6 Result card — событие
- Та же карточка + вертикальная полоса 4 px (5 px вечером) цвета `--ld-event` по левому краю (внутри border-radius, через overflow hidden).
- Иконка `ti ti-ticket` цвета `--ld-event` в правом верхнем углу вместо/рядом с сердцем (сердце тоже доступно).
- Метаданные: «Опера · сегодня 19:00 · 1.2 км». Если старт < 3 ч — бейдж «Начнётся через 1:20» (`--ld-glow-soft` вечером / `--ld-primary-soft` днём).
- Бейдж цены: «Билеты от 30 ₾». CTA «Билеты» появляется только на detail-странице.

### 6.7 Кнопки
| Вариант | Стиль | Использование |
|---|---|---|
| Primary | заливка `--ld-primary`, текст `--ld-on-primary`, 48 px, радиус 14 | 1 шт. на экран: «Показать места» в фильтрах, «Начать» в онбординге |
| Secondary | `--ld-surface`, бордер 1 px в тоне primary (день #E8C79E / вечер #F0B4BE), текст `--ld-on-primary-soft` | «Сбросить», «Поделиться» |
| Ghost | без фона, текст `--ld-primary` | текстовые действия в sheets |
| Icon | 44×44, круглая, `--ld-surface-2` | назад, закрыть, сердце на detail |

### 6.8 Filter sheet (bottom drawer)
- Радиус верха 24, drag-handle 36×4 px `--ld-border`.
- Секции: Тип (сегмент-контрол Все/Места/События — pill-переключатель на `--ld-surface-2`, активный сегмент `--ld-surface` + тень), Радиус (slider, трек `--ld-border`, заливка и thumb `--ld-primary`), Интересы (grid чипов 2 колонки), Компания (4 чипа с иконками), Питомец (toggle).
- Внизу sticky-ряд: Ghost «Сбросить» + Primary «Показать (24)». Число результатов живое.

### 6.9 Detail page
- Без hero-фото в MVP: цветная плашка-заголовок высотой ~120 px на `--ld-primary-soft` (место) / `--ld-event-soft` (событие) с крупной категорийной иконкой Tabler 40 px и кнопками назад/сердце/поделиться поверх.
- Далее: title 20/700, метаданные, полный ряд explanations, блок «Почему это вам» (все причины скоринга словами), часы работы (сегодня раскрыто, неделя — collapsible), рейтинг, адрес.
- Действия-ряд (иконка + подпись, 4 шт.): Маршрут (`ti ti-route`, deeplink в Google Maps), Сохранить, Поделиться (`navigator.share`), Скрыть.
- Для события: sticky-кнопка Primary «Билеты от 30 ₾» внизу (единственный primary на экране).

### 6.10 Onboarding (3 шага, каждый — один экран)
1. Интересы: grid чипов, минимум 1. 2. Компания + питомец. 3. Локация: «Разрешить GPS» primary / «Ввести координаты» ghost.
- Прогресс — 3 точки. Кнопка «Пропустить» вверху справа (ghost) на каждом шаге — ленивый путь обязан существовать.
- Display-заголовки Unbounded, по одному вопросу на экран.

### 6.11 Skeleton / Empty / Error
- Skeleton: форма карточки, блоки `--ld-surface-2`, shimmer 1.2 s (отключается при reduced-motion). Показывать 5 штук.
- Empty (нет результатов): иконка `ti ti-zzz` 40 px `--ld-text-3`, заголовок «Слишком лениво даже для нас», текст «Расширьте радиус или снимите фильтр», кнопка Secondary «Радиус +50 %» — одно действие, решающее проблему.
- Error API: «Не получилось загрузить. Проверьте сеть» + Secondary «Повторить». Без кодов ошибок в UI.

### 6.12 Toast / feedback
- Снизу над tab bar, `--ld-text` фон / текст `--ld-bg` (инверсия), радиус 12, 2.5 s.
- «Сохранено» с ghost-действием «Отменить». «Скрыто · Отменить» — обязательно с undo, скрытие деструктивно.

## 7. Доступность и качество

- Контраст по разделу 3; фокус-кольцо 2 px `--ld-primary` + offset 2 px для клавиатуры/скринридеров.
- `aria-label` на всех icon-only кнопках; статусы «Открыто/Закрыто» — текстом, не только цветом.
- Локали ru/en/ka: чипы и бейджи рассчитывать на +30 % длины текста (грузинский шире), никаких фиксированных ширин у чипов.
- Тап-таргеты ≥ 44 px, между соседними интерактивами ≥ 8 px.
- Lighthouse-цель: Performance ≥ 90, Accessibility ≥ 95 на mid-range Android.

## 8. Порядок внедрения

1. Токены трёх тем в `styles.scss` + `theme` signal в ProfileStore + авто-переключение по времени.
2. Рефактор существующих компонентов на `--ld-*` переменные (карточка, чипы, tab bar).
3. Greeting header + вечерний набор mood-чипов.
4. Filter sheet и detail по спеке.
5. Onboarding, empty/error states, toasts.
