# Как тестировать персонализацию на фронте

*Руководство для ручного тестирования на localhost:4200*

---

## Быстрый старт

```
1. Запустить бек:   GEMINI_API_KEY=xxx npx nx serve api
2. Запустить фронт: npx nx serve lazy-day
3. Открыть:         http://localhost:4200
```

---

## Что проверять и где

### 1. Лента (Discover page)

**URL:** `http://localhost:4200/discover`

**Что видно:**
- Карточки мест/событий с 3 слотами (title, meta, status)
- whyLabel под мета-строкой (появляется после 3+ сигналов)
- Полоска слева: оранжевая (place) / фиолетовая (event)
- "Часы не подтверждены" для мест без свежих opening hours

**Как тестировать персонализацию:**
1. Открой ленту — все места отсортированы по base score (distance + interest). whyLabel пустой.
2. Нажми ♡ (сохрани) на 3 upscale места (бары, рестораны с высоким рейтингом).
3. Перезагрузи страницу — порядок должен измениться. Upscale места поднимутся.
4. После 3+ сигналов (save/route/share) — на карточках появится "Ваш вайб: cozy" или подобное.

**Ожидание при cold start (0 сигналов):**
- whyLabel пустой для всех (нет данных для персонализации)
- Только "Рядом с вами" (<500м) или "Новое место рядом" (epsilon slot)
- Это BY DESIGN — не баг

---

### 2. Профиль вкуса (Settings)

**URL:** `http://localhost:4200/settings`

**Что видно:**
- Секция "Ваш вкус" — появляется после 1+ сигнала
- Позитивные фасеты как чипы (можно удалить нажатием ×)
- Негативные фасеты (если были hide) — тоже чипы
- Кнопка "Сбросить вкус"

**Как тестировать:**
1. Сохрани 3 винных бара → перейди в Settings
2. Должно появиться: "Кажется, вам нравится: wine_bar, cozy, romantic..."
3. Нажми × на одном чипе → он исчезает → вернись в ленту → порядок изменился
4. "Сбросить вкус" → все чипы исчезают → лента вернулась к base scoring

---

### 3. "Как это работает" (Settings)

**URL:** `http://localhost:4200/settings` → кнопка "Как это работает"

Открывается bottomsheet с 5 пунктами:
- Учимся по тапам/сохранениям/маршрутам
- Подмешиваем новое
- Скрытые не появятся
- Вкус можно поправить
- Всё на устройстве

---

### 4. "Реши за меня" (Discover)

**URL:** `http://localhost:4200/discover` → кнопка 🧭 (пульсирует)

**Что проверять:**
- Три разных предложения (не повторяются)
- Если есть события — минимум 1 из 3 = событие
- "Другой вариант" показывает ДРУГОЕ место (MMR diversity)
- После hide → это место не появится снова

### 5. Score Decomposition (explain endpoint)

**НЕ на фронте — через curl/DevTools.**

```bash
curl -s -X POST "http://localhost:3000/v1/recommendations/explain" \
  -H "Content-Type: application/json" \
  -d '{
    "lat": 41.749, "lng": 44.786, "radiusM": 5000,
    "timeWindow": {"from": "2026-07-19T14:00:00Z", "to": "2026-07-19T23:00:00Z"},
    "profile": {"interests": {"food": 1}},
    "hiddenIds": [], "locale": "ru"
  }' | python -m json.tool
```

**Что видно в ответе:**
```json
{
  "profileSnapshot": {
    "facet_weights": {},
    "signal_count": 0,
    "w_personal": 0
  },
  "results": [
    {
      "name": "McDonald's",
      "rank": 1,
      "finalScore": 0.923,
      "components": {
        "interest": 0.45,
        "distance": 0.243,
        "time": 0.15,
        "quality": 0.05,
        "source": 0.03,
        "personalization": 0,
        "priceBoost": 0
      },
      "facets": {
        "cuisine": ["burgers"],
        "format": ["fast_food"],
        "atmosphere": ["casual", "family_friendly"],
        "occasion": ["quick_stop"]
      }
    }
  ]
}
```

**Как использовать:**
- `personalization: 0` = cold start, нет сигналов
- `personalization: 0.14` = профиль работает, cosine match высокий
- Сравни `components` двух мест чтобы понять ПОЧЕМУ одно выше другого

## Сценарии тестирования

### Сценарий 1: Upscale Foodie
1. Открой discover с фильтром "Поесть"
2. Сохрани 3 ресторана с высоким рейтингом (★4.5+)
3. Перезагрузи → эти и похожие рестораны должны подняться
4. McDonald's/фастфуд должны опуститься (но не исчезнуть)
5. В Settings → "Ваш вкус" → должны быть "upscale", "fine_dining"

### Сценарий 2: Nightlife
1. Выбери фильтр "Ночная жизнь"
2. Нажми "Маршрут" на 2 барах
3. Перезагрузи → бары с live_music, lively атмосферой поднимутся
4. Тихие музеи/парки НЕ должны появляться в выдаче (interest filter)

### Сценарий 3: Hide и негатив
1. Скрой (hide) 2 фастфуд места
2. Они должны исчезнуть из ленты сразу
3. Другие фастфуд места НЕ должны исчезнуть (нужно ≥2 hide на тот же фасет)
4. Скрой ещё 1 фастфуд → теперь `fast_food` фасет оштрафован → остальные фастфуд опустятся

### Сценарий 4: Cold Start → тёплый
1. Открой в incognito (чистый профиль)
2. whyLabel пустой, сортировка по distance + quality
3. Сохрани 1 место → перезагрузи → минимальное изменение (w_personal ≈ 0.013)
4. Сохрани ещё 4 → перезагрузи → заметное изменение (w_personal ≈ 0.067)
5. В Settings → "Ваш вкус" → показывает фасеты

### Сценарий 5: Сброс
1. Накопи профиль (5+ сохранений)
2. Settings → "Сбросить вкус"
3. Перезагрузи → лента вернулась к base scoring (как cold start)

## Частые вопросы

**Q: Почему whyLabel не появляется?**
A: Нужно минимум 3 сигнала (save/route/share). И cosine match > 0.5. На cold start — пустой by design.

**Q: Почему порядок не меняется после одного save?**
A: w_personal = 0.013 после 1 сигнала (из макс 0.20). Минимальное влияние. После 5+ — заметно.

**Q: Почему McDonald's всё ещё показывается?**
A: Он никогда не был скрыт (hide). Personalization понижает score, но category floor 10% защищает.

**Q: Почему "Часы не подтверждены" у многих мест?**
A: enriched_at > 30 дней или null. На проде будет свежее после refresh cron.

**Q: Как увидеть score breakdown?**
A: Через `POST /v1/recommendations/explain` (curl). Dev dashboard (`/dev/reco-lab`) — будущее.
